The local-first data studio with a built-in AI assistant.

Duckle is an open-source desktop ETL / ELT studio. Drag a pipeline onto the canvas, or describe it in plain English to Duckie - the on-device AI - and run it at native columnar speed through DuckDB. 329 components, a scheduler, and an MCP server, in a ~65 MB single-file app.

Part of the DuckDB ecosystem

Duckle turns DuckDB into a visual, embeddable ETL platform.

Every pipeline compiles to SQL and executes through DuckDB - vectorized, columnar, local. Duckle sits in the ingestion + transformation layer of the DuckDB stack, next to tools like dlt, dbt, SQLMesh and Evidence, and reads the wider DuckDB family natively: DuckLake tables, MotherDuck, the Quack remote protocol, and the community extensions.

Use cases

From a CSV cleanup to a RAG pipeline - in a few nodes.

Warehouse loading & replication

Read from a production database and upsert into your warehouse on a schedule, with change-data-capture and delete propagation.

  • Universal MERGE / upsert across every relational and warehouse sink
  • DuckLake CDC change-feed + watermark incremental loads
  • Attach a cron schedule to run nightly, unattended
# Postgres → Snowflake nightly upsert
src.postgres → snk.snowflake
mode=upsert, conflict=event_id
schedule: 0 2 * * *
# Chunk, redact, embed, dedupe, store
src.s3 → xf.ai.chunk → xf.ai.pii
→ xf.ai.embed → xf.ai.dedupe
→ snk.pgvector

Clean data before it reaches your AI

RAG indexes quietly fill with duplicates, nulls and PII. Scrub data before it lands in a vector store - locally.

  • Chunk → redact PII → embed → semantic-dedupe, all on device
  • Hybrid retrieval: vector similarity (VSS) + full-text (BM25), no model API
  • Working sinks for pgvector, Qdrant, Weaviate, Milvus, Pinecone

Files & object storage at native speed

Glob thousands of files from S3, parse and reshape, and write Hive-partitioned Parquet - vectorized end to end.

  • Autodetect formats, including gzipped JSON and globbed paths
  • Partition-by writes with a fail-fast guard against file explosions
  • A 16-node, multi-branch run finishes in a few seconds
# S3 logs → partitioned Parquet
src.s3 (glob *.json.gz)
→ xf.addcol (event_date)
→ snk.parquet (partitionBy=event_date)
Meet Duckie

Describe what you need. Duckie writes the pipeline.

The assistant in the right sidebar is powered by Qwen 2.5 Coder 1.5B through llama.cpp, downloaded once (~1.1 GB) and then run entirely on your CPU.

  • Truly local - runs as a subprocess on 127.0.0.1. Disconnect wifi; it keeps working.
  • One-click insert - Duckie's JSON populates the canvas with positioned, wired nodes.
  • Bring your own model - point it at Ollama, OpenAI, or anything OpenAI-compatible.
  • Sandboxed - the model has no filesystem, network, or tool access. It only emits pipeline JSON.
